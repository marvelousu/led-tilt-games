# ジャイロ操作 調整ガイド

Arduino ゲーム機のジャイロ（LSM6DSV16X）操作系の調整方法。

**調整したい値はすべて `config.h` の「ジャイロ操作 調整パラメータ」区画にある。**
実機で感度を変えたいときは、そこの数値だけを書き換えれば済む。

---

## ジャイロが提供する 3 つの入力

| 入力 | 用途 | API (`gyro.h`) |
|------|------|----------------|
| 傾き (tilt)     | メニュー選択、各ゲームの移動・重力 | `gyro_tilt_x` / `gyro_tilt_y` |
| シェイク (shake) | 砂ゲームの砂補充・増殖             | `gyro_shake_consumed` |
| 縦持ち回転 (rotate) | DODGE 開始時の縦持ち検知        | `gyro_rotate_reset` / `gyro_rotate_detected` |

いずれも `gyro.cpp` が生 I2C で加速度・角速度を読み、閾値で判定する。

---

## 調整パラメータ（`config.h`）

| 名前 | 既定値 | 役割 | 下げると | 上げると |
|------|--------|------|----------|----------|
| `IMU_TILT_THRESHOLD_MG`  | 300   | 傾き入力が成立する傾き量 (mg) | 敏感になる | 鈍感になる |
| `IMU_SHAKE_THRESHOLD_MG` | 2000  | シェイク成立の加速度 (mg) | 弱い振りで反応 | 強い振りが必要 |
| `IMU_SHAKE_RELEASE_PCT`  | 75    | シェイク解除のしきい割合 (%) | 連続シェイク不可 | 連続しやすい |
| `GYRO_ROT_DEADZONE`      | 300   | 回転検知の静止ノイズ除去 (raw) | 微動も拾う | ゆっくりだと拾わない |
| `GYRO_ROT_THRESHOLD`     | 60000 | 90°回転と判定する積算量 | 少しの回転で成立 | しっかり回す必要 |

---

## 症状別 調整早見表

| 症状 | 調整 |
|------|------|
| 傾けても反応が鈍い | `IMU_TILT_THRESHOLD_MG` を下げる |
| 平らに置いても勝手に動く | `IMU_TILT_THRESHOLD_MG` を上げる |
| 軽く振っただけで誤検出する | `IMU_SHAKE_THRESHOLD_MG` を上げる |
| 強く振らないとシェイクしない | `IMU_SHAKE_THRESHOLD_MG` を下げる |
| DODGE で回しても始まらない | `GYRO_ROT_THRESHOLD` を下げる |
| DODGE が少し動かしただけで始まる | `GYRO_ROT_THRESHOLD` を上げる |

---

## 仕組みの補足

- **傾き**: 加速度 ax / ay が閾値 (mg) を超えたら方向 ±1 を返す。
- **シェイク**: 加速度ベクトルの大きさが閾値を超えたら成立。一度成立すると
  `IMU_SHAKE_RELEASE_PCT` の割合まで下がるまで再成立しない（ヒステリシス）。
- **縦持ち回転**: ジャイロ Z 軸（角速度）を毎フレーム積算する。
  `GYRO_ROT_DEADZONE` 以下のサンプルは無視（静止時のドリフト除去）、
  累積が `GYRO_ROT_THRESHOLD` に達したら「回転完了」と判定する。
  - 積算量はジャイロのフルスケール（`gyro.cpp` の `CTRL6_VAL`）に依存する。
    `CTRL6_VAL` を変更したら `GYRO_ROT_THRESHOLD` も再調整が必要。

---

## ハードウェア設定（`gyro.cpp`）

レジスタ値 `CTRL1_VAL` / `CTRL2_VAL` / `CTRL6_VAL` などはセンサーの動作モード
（加速度・ジャイロの ODR / フルスケール）設定で、通常は変更不要。
変更にはデータシートの参照が必要。
`GYRO_ROT_THRESHOLD` は `CTRL6_VAL` のフルスケール設定に依存するので、
レジスタを書き換えたら閾値も再調整する。
