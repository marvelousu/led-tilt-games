# 音楽（BGM / SE）編集ガイド

ゲーム機の音は単音ブザー（ピエゾ）。すべての曲・効果音は `buzzer.cpp` の
`Note` 配列で定義する。試聴は `music_preview.html`（ブラウザのみ、Arduino 不要）。

---

## 音楽の置き場所（すべて buzzer.cpp）

各曲は次の形で書かれている:

```c
static const Note <NAME>_SEQ[] PROGMEM = {
    {周波数Hz, 長さms}, {周波数Hz, 長さms}, ...
};
```

### BGM

| 配列 | 使われる場面 |
|------|--------------|
| `BGM_TAG_SEQ`   | 鬼ごっこ（skippy 系） |
| `BGM_DODGE_SEQ` | 落下よけ（bright 系） |
| `BGM_MENU_SEQ`  | メニュー |

砂あそびは BGM なし（SE のみ）。

どのゲームがどの BGM を鳴らすかは `game_tag.cpp` / `game_dodge.cpp` の
`buzzer_play_bgm(...)` 呼び出しで決まる。

### SE（効果音）

| 配列 | 用途 |
|------|------|
| `SE_CATCH_SEQ`    | 鬼ごっこで捕まえた |
| `SE_GAMEOVER_SEQ` | ゲームオーバー |
| `SE_START_SEQ`    | ゲーム開始 |
| `SE_BOOT_SEQ`     | 起動音 |
| `SE_WARN_SEQ`     | 警告（残り時間など） |
| `SE_CLEAR_SEQ`    | クリア |
| `SE_HIT_SEQ`      | 衝突 |
| `SE_MENU_SEQ`     | メニュー遷移 |

---

## Note の書式

- `{周波数Hz, 長さms}`
- 周波数 `0` は休符（その長さぶん無音）
- 配列の長さは `sizeof` で自動計算される → 音を増減しても他の修正は不要

---

## ★最重要の制約：周波数は 392Hz 以上にする

Arduino Nano R4 の `tone()` は **約 366Hz 未満でタイマー分周器が切り替わり、
音程が外れる**（実機で確認済み）。

**すべての音符を 392Hz 以上にすること**（休符の `0` は可）。

> 既知の未修正：`SE_GAMEOVER_SEQ`（349 / 294 / 247Hz）と `SE_HIT_SEQ`（262Hz）は
> 366Hz 未満の音を含むため、実機では音程がずれる。気になる場合は音を上げる。

---

## 試聴（Arduino 不要）

`music_preview.html` をブラウザで開き、`buzzer.cpp` をドラッグ&ドロップ
（またはファイル選択）→ 曲名ボタンをクリックで再生。

- `piezo sim` ON ：実機ブザーの鳴り方を近似（低音が痩せる）
- `piezo sim` OFF：本来の音程
- OS 非依存（Windows / Mac / Linux のブラウザで動く）

`buzzer.cpp` を編集 → ブラウザでファイルを選び直す、で即試聴できる。

---

## 候補曲を試す（bgm_candidates.txt）

`bgm_candidates.txt` に `<NAME>_SEQ[] PROGMEM = { ... };` の形で曲を書くと、
`music_preview.html` で `buzzer.cpp` と一緒に読み込んで試聴できる。
このファイルはビルドに含まれない（試聴専用）。
採用が決まったら、その中身を `buzzer.cpp` の対象 BGM 配列へ移す。

---

## 長さ・テンポ・同時発音

- 1音の長さは `Note` の ms 値。曲を長くするには音符を増やす
- BGM はゲーム中ループ再生される
- 単音ブザーのため**同時発音はできない**。SE 再生中は BGM が一時中断し、
  SE が終わると BGM が再開する
